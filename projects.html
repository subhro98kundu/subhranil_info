Amazon Clone Website (Full Stack Web Development)

This is my first full stack web development project. The website is user-interactive. While working on this project I learnt DBMS and how proper queries are made. I learnt how important it is to plan the working flow of a website before starting to code it. This project taught me how to create efficient databases and tables. While planning I divided the project into small modules and kept working on them one by one. This helped me in keeping a track of my progress and made sure that i don't miss out any module. Breaking the project into small modules also made any modification easier. The project was all about planning, writing and debugging thousands of lines of code. I won't bore you with each and every details the code is available on my github, you can check it here. But, here I will like to cover important details in my project.
  • I built a Sign Up module for the website that allows new users to register in the website. While building this module I learnt about registering a new user in the database and how to handle user sessions on the client side.
  • After that I built a login module. This allowed already registerd users to log into their respective accounts using their email and password. I also took care of security.
  • Then I created a table for products in the database and I had to plan a lot about the columns of the tables to make it efficient. and after making the table I displayed the products on the web page in various categories as they do in amazon by making queries based on different categories. Well i didn' have a big enough table. So unfortunately it has only four major categories.
  • Then I coded the Add to Cart module, where if a user is logged in he can add items to his cart. And the items will remain in his/her cart until he/she removes it or purchases it. For this I had to made another table for cart but this table only had 4 columns cart id(primary key), user id, product id and quantity and I linked the table to the products table and the users table with the product id and user id respectively.
  • Then I created the Purchase Module where the user is allowed to add a delivery address or chose from the ones added by him/her earlier. For this I had to create a table for addresses linked with the users table by user id. I also implemented a dummy payment gateway in the website. And once the order is placed the user can check the status of his order from the orders tab.
  • After that developed the Review Module where a user can review the products he ordered.
 Cloning each and every feature of amazon was a tough challenge. But as my first project on full stack web development, I tried my best clone maximum of the main features.

Zomato Clone Website (Full Stack Web Development)

This is a full stack web development project. The website is user-interactive. While working on this project I improved my knowledge in DBMS. I did a thorough planning of the working flow of the website before starting to code it down. In this project I improved my UI skills and implemented a better working UI than my amazon project. While planning I divided the project into small modules and kept working on them one by one. This time my project was more organized than the Amazon one. This helped me in keeping a track of my progress and made sure that i don't miss out any module. Breaking the project into small modules also made any modification easier. The project was all about planning, writing and debugging thousands of lines of code. I won't bore you with each and every details the code is available on my github, you can check it here. But, here I will like to cover important details in my project.
  • I built a Sign Up module for the website that allows new users to register in the website. While building this module I learnt about registering a new user in the database and how to handle user sessions on the client side.
  • After that I built a login module. This allowed already registerd users to log into their respective accounts using their email and password. I also took care of security.
  • Then I created two separate tables one for restaurants and one for products in the database and I had to plan a lot about the columns of the tables to make it efficient. After making the table I displayed the restaurants on the web page in various categories as they do in zomato by making queries based on different categories. Well this time I have a big enough table. So fortunately I could cover a lot of categories.
  • Then I coded the Add to Cart module, where if a user is logged in he/she can add food items(menu) to his/her cart. And the items will remain in his/her cart until he/she removes it or purchases it. For this I had to made another table for cart but this table only had 4 columns cart id(primary key), user id, menu id and quantity and I linked the table to the menu table and the users table with the product id and user id respectively.
